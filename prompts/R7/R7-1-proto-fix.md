# R7-1 — Fix IBC Proto Registration

## Goal

Fix the proto descriptor registration issue in `x/ibcratelimit` and `x/icaauth` that causes
`app` and `cross_stack` tests to panic with:
```
cannot find method descriptor zerone.ibcratelimit.v1.Msg.AddRateLimit
```

## Working Directory

`/Users/yuai/Desktop/Zerone`

## Problem

The `tx_grpc.pb.go` files were generated with `protoc-gen-go-grpc` (newer style) but the
`tx.pb.go` files use `gogoproto` style. Cosmos SDK v0.50 uses `gogoproto` for service
registration — the `_Msg_serviceDesc` service descriptor must be present in `tx.pb.go`
for `RegisterMsgServer` to work with `baseapp.MsgServiceRouter`.

## Steps

1. **Diagnose** — check how other working modules (e.g. `x/emergency`, `x/billing`) generate
   their proto code vs ibcratelimit/icaauth. Compare `tx.pb.go` files — working ones should
   have `_Msg_serviceDesc` and `RegisterMsgServer` in the same file.

2. **Regenerate protos** — use the same proto generation tooling as the rest of the codebase.
   Check `Makefile` or `scripts/` for the `proto-gen` command. Run it for:
   - `proto/zerone/ibcratelimit/v1/*.proto`
   - `proto/zerone/icaauth/v1/*.proto`

3. **If no proto-gen script exists**, check how other modules' `tx.pb.go` were generated
   (look at the `// Code generated by` header). Match that tooling exactly.

4. **Verify** — run:
   ```bash
   go test ./app/... ./tests/cross_stack/...
   ```
   Both must pass (no panics).

5. **Run full suite** — `go test ./...` — all 27+ test packages must pass.

## Reference

- Working module for comparison: `x/emergency/types/tx.pb.go` (has gogoproto service descriptor)
- Broken: `x/ibcratelimit/types/tx.pb.go` + `tx_grpc.pb.go` (separate files, missing descriptor)
- Cosmos SDK v0.50 proto pattern: single `tx.pb.go` with both message types AND gRPC service

## Constraints

- Do NOT change any module logic — this is a proto tooling fix only
- Match the exact proto generation pattern used by all other Zerone modules
- If the proto files themselves need `option (gogoproto.*)` annotations, add them
