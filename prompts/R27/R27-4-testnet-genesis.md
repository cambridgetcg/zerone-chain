# R27-4 — Public Testnet Genesis Configuration

## Context

The localnet genesis is auto-generated by `scripts/localnet.sh` with defaults optimized for local testing (short block times, small validator set, test keyring). A public testnet needs different parameters, a proper chain-id, and a genesis file that external validators can use to join.

## Task

### 1. Create Testnet Genesis Generator

Create `scripts/testnet-genesis.sh` that produces a genesis.json for `zerone-testnet-1`:

```bash
#!/bin/bash
# Generates genesis.json for zerone-testnet-1
# Usage: ./scripts/testnet-genesis.sh [output-dir]
```

### 2. Chain Parameters

Configure params that differ from localnet:

**Consensus:**
- `chain_id`: `zerone-testnet-1`
- `block_time`: target 6s (CometBFT `timeout_commit`)
- `max_gas`: 50,000,000 per block

**Knowledge Module:**
- `commit_phase_blocks`: 50 (~5 min)
- `reveal_phase_blocks`: 50 (~5 min)
- `min_claim_stake`: 1,000,000 uzrn (1 ZRN)
- `min_review_fee`: 100,000 uzrn (0.1 ZRN)
- `metabolism_epoch_blocks`: 1,000 (~100 min, not 10K)

**Qualification:**
- `min_stake_for_qualification`: 100,000,000 uzrn (100 ZRN)
- `min_verifications_for_track_record`: 50 (lower than mainnet 100 for testnet iteration)
- `min_accuracy_for_track_record`: 7,500 (75%, lower than mainnet 80%)

**Partnerships:**
- `coercion_review_blocks`: 100 (~10 min)
- `consensus_op_expiry_blocks`: 100 (~10 min, not 22)

**Research:**
- `review_period_blocks`: 500 (~50 min)
- `min_reviewer_count`: 2 (lower for testnet)
- `bounty_fulfillment_period_blocks`: 1,000 (~100 min)

**Vesting Rewards:**
- Revenue split: 55/22/19.67/3.33 (same as mainnet)
- `base_block_reward`: set appropriately for testnet inflation

**Governance:**
- `voting_period`: 86,400s (1 day, not 14 days)
- `min_deposit`: 10,000,000 uzrn (10 ZRN)
- `quorum`: 0.25 (lower for small validator set)

**Staking:**
- `unbonding_time`: 86,400s (1 day)
- `max_validators`: 20
- `min_self_delegation`: 1,000,000 uzrn (1 ZRN)

### 3. Genesis Accounts

The genesis should include:
- **Faucet account** — pre-funded with testnet tokens for distribution
- **No other pre-funded accounts** — participants earn through registration + faucet
- **777 axiom facts** — same as localnet (foundational knowledge base)

### 4. Validator Onboarding

Create `docs/testnet-validator-guide.md` covering:

1. **Hardware requirements** (minimal for testnet)
2. **Install zeroned** (from release binary or Docker)
3. **Initialize node** — `zeroned init <moniker> --chain-id zerone-testnet-1`
4. **Download genesis** — where to get genesis.json
5. **Set persistent peers** — seed node addresses
6. **Start node** — `zeroned start`
7. **Create validator** — `zeroned tx staking create-validator ...`
8. **Register account** — `zeroned tx zerone-auth register-account <type>`
9. **Qualify for domains** — `zeroned tx qualification qualify-stake ...`

### 5. Seed Node Configuration

Set up at least one seed node (can be on zerone VPS or Yu's machine):
- Document the seed node's peer address
- Configure `seeds` in genesis or as a well-known endpoint
- Consider using the existing zerone VPS (80.78.19.135) as seed

### 6. Genesis Validation

```bash
# Validate genesis
zeroned genesis validate genesis.json

# Test: start a single node from this genesis
zeroned start --home /tmp/testnet-test
# Verify it produces blocks
```
